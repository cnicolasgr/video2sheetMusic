# video2sheetMusic

A command-line tool to extract sheet music scans from YouTube videos. The program downloads a video, extracts the relevant score frames, stitches them back into a continuous sheet, cleans the output, and saves the result as PNG and PDF.

## What it does

`video2sheetMusic` is designed for videos that show sheet music moving across the screen. It is not a music transcription tool. Instead, it reconstructs the sheet music as an image by:

- downloading the YouTube video using `yt-dlp`
- extracting the video frames that contain significant changes
- cropping and aligning the sheet music area
- stacking the images into a long sheet
- cleaning the result with adaptive thresholding
- exporting a multipage PDF formatted to A4 proportions

## Dependencies

The project uses the following Python libraries:

- `yt-dlp`
- `scikit-image`
- `opencv-python`
- `numpy`
- `tqdm`
- `Pillow`

## Installation

1. Create and activate a Python virtual environment:

```bash
python -m venv myenv
source myenv/bin/activate
```

2. Install the dependencies:

```bash
pip install -r requirements.txt
```

If you already have the included `myenv`, activate it instead:

```bash
source myenv/bin/activate
```

## Usage

Run the main script with a YouTube video URL:

```bash
python main.py https://www.youtube.com/watch\?v\=lTpl_6Moo24
```

The default output directory is `data/output/`.

### Optional arguments

- `--output_dir`, `-o`: destination folder for final results
- `--skip-frames`, `-s`: number of frames to skip between comparisons
- `--ssim-threshold`, `-t`: SSIM threshold used to decide whether a new frame should be saved
- `--system-spacing`, `-sp`: vertical pixel spacing added between stitched systems

Example:

```bash
python main.py https://www.youtube.com/watch\?v\=lTpl_6Moo24 -o data/output -s 5 -t 0.88 -sp 80
```

## What gets produced

After a successful run, the `data/output/` folder contains:

- `stitched_sheet.png`: the vertically joined sheet image
- `cleaned_sheet.png`: the binarized version of the stitched image
- `{video_title}.pdf`: the final PDF file generated from the cleaned image

The script also stores downloaded video files in `data/input_video/` and temporary extracted frames in `data/temp_frames/`.

## Notes

- The tool expects sheet music to be visible in the video. It tries to detect the music area automatically by finding the largest light-colored contour in the middle frame.
- If the automatic crop fails, `opencv` will open a window and ask you to select the region of interest manually.
- The PDF output is generated by slicing the long image into A4-like chunks and preserving white gaps as page breaks.

## Project structure

- `main.py`: entry point that coordinates download, extraction, stitching, cleaning, and PDF generation
- `src/video_engine.py`: downloads video and extracts distinct score frames
- `src/image_stitcher.py`: aligns and concatenates frame images into one long sheet
- `src/image_cleaner.py`: binarizes the stitched image and converts it into PDF
- `src/config.py`: default parameters and data path configuration

## Troubleshooting

- For very noisy videos, lower the `--ssim-threshold` or adjust `--skip-frames` to capture more frames.
- This code still doesn't handle "scrolling score" videos
